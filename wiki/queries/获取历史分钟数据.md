# 获取历史分钟数据

> **问题**：怎么获取历史分钟数据？

## 回答

使用 `stk_mins` 接口获取股票历史分钟级行情数据。

### stk_mins — 股票分钟行情

支持 1/5/15/30/60 分钟五种频率，单次最大返回 8000 行。

```python
import tushare as ts
pro = ts.pro_api('your_token')

# 获取 1 分钟行情
df = pro.stk_mins(ts_code='000001.SZ', freq='1min',
                  start_date='2024-01-15 09:00:00',
                  end_date='2024-01-15 15:00:00')

# 获取 5 分钟行情
df = pro.stk_mins(ts_code='000001.SZ', freq='5min',
                  start_date='2024-01-15 09:00:00',
                  end_date='2024-01-15 15:00:00')

# 获取 60 分钟行情
df = pro.stk_mins(ts_code='600519.SH', freq='60min',
                  start_date='2024-01-01 09:00:00',
                  end_date='2024-01-31 15:00:00')
```

### 权限要求

- 需要单独开通分钟数据权限（与积分独立）
- 120 积分可试调 2 次，正式使用需单独申请权限

### 注意事项

1. **freq 参数**：必须从 `1min`/`5min`/`15min`/`30min`/`60min` 中选择
2. **时间范围**：start_date 和 end_date 为 datetime 格式，如 `'2024-01-15 09:30:00'`
3. **数据量控制**：单次最大 8000 行，可通过日期范围分批提取
4. **复权处理**：分钟数据为不复权原始数据，如需复权需自行计算

### 相关接口对比

| 接口 | 数据类型 | 频率 | 实时性 |
|------|----------|------|--------|
| `stk_mins` | 历史 | 1/5/15/30/60min | 延迟 |
| `rt_min` | 实时 | 1-60min | 实时 |
| `opt_mins` | 期权历史 | 1/5/15/30/60min | 延迟 |
