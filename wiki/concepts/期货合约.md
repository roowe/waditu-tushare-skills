# 期货合约

> 期货合约是标准化的远期交易协议，约定在未来特定日期以特定价格买卖标的资产。每个期货品种同时存在多个月份合约（如 CU2501、CU2502），其中持仓量最大的合约为主力合约，拼接主力合约形成连续合约用于长期分析。

## 核心概念

| 概念 | 说明 | 关键字段/接口 |
|------|------|--------------|
| 期货合约(月合约) | 标准化合约，按交割月份区分 | `fut_basic`（ts_code, symbol, exchange） |
| 主力合约 | 持仓量最大的月合约，代表品种主力流动性 | `fut_mapping`（mapping_ts_code） |
| 连续合约 | 将每日主力合约拼接形成的价格序列 | `fut_mapping`（ts_code 如 TF.CFX） |
| 涨跌停价格 | 交易所每日公布的各合约涨跌停板价格 | `ft_limit`（up_limit, down_limit, m_ratio） |
| 仓单 | 交割仓库/厂库签发的标准仓单，反映可交割量 | `fut_wsr`（vol, vol_chg, warehouse, pd） |
| 结算参数 | 每日结算价、手续费率、保证金率等 | `fut_settle`（settle, trading_fee_rate, long_margin_rate） |
| 合约乘数/交易单位 | 每手合约的规模 | `multiplier`, `per_unit`, `trade_unit` |

## 详细说明

### 合约信息与交易所

`fut_basic` 接口提供全市场期货合约的基础信息，覆盖六大交易所：CFFEX（中金所）、DCE（大商所）、CZCE（郑商所）、SHFE（上期所）、INE（上海国际能源交易中心）、GFEX（广州期货交易所）。每张合约包含 ts_code（合约代码）、symbol（交易标识）、name（中文简称）、list_date（上市日期）、delist_date（最后交易日）等信息。通过 `fut_type='2'` 可筛选主力与连续合约。

### 主力合约与连续合约

主力合约是某品种持仓量最大的月合约，通常也是成交最活跃的合约。`fut_mapping` 接口记录了每日主力合约/连续合约与实际月合约的映射关系。例如 TF.CFX（五年期国债期货连续合约）在 20190812 之前对应 TF1909.CFX，之后切换到 TF1912.CFX。利用此映射可以构建无跳价的连续价格序列。

### 涨跌停价格

`ft_limit` 接口提供每日盘前公布的各合约涨跌停价格（up_limit/down_limit）和最低交易保证金率（m_ratio）。数据从 2005 年开始，支持按合约代码（cont）、交易所（exchange）等维度筛选。保证金率区分投机和套保持仓。

### 仓单日报

仓单是期货交割的重要凭证。`fut_wsr` 接口按仓库/厂库维度记录每日仓单数量变化（pre_vol -> vol -> vol_chg），还包含地区（area）、品牌（brand）、产地（place）、升贴水（pd）等属性。仓单增减反映可交割品的注册/注销情况，是判断交割压力的重要参考。

### 结算参数

`fut_settle` 接口提供每日结算价（settle）、交易手续费率（trading_fee_rate）、交割手续费（delivery_fee），以及按持仓性质区分的保证金率：买套保（b_hedging_margin_rate）、卖套保（s_hedging_margin_rate）、买投机（long_margin_rate）、卖投机（short_margin_rate）。近月合约的保证金率通常高于远月。
