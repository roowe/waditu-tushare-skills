> 存活偏差（Survivorship Bias）和前视偏差（Look-ahead Bias）是量化研究中两个最常见的系统性数据陷阱。存活偏差导致回测仅基于存续股票而忽略已退市股票，从而高估策略收益；前视偏差则在回测时使用了当时不可得的信息，造成虚高的回测业绩。

## 核心概念

| 概念 | 英文 | 说明 | 防范方法 |
|------|------|------|---------|
| 存活偏差 | Survivorship Bias | 回测时仅使用当前仍在上市的股票，遗漏已退市股票，导致策略收益被高估 | 使用全量股票列表（含退市股） |
| 前视偏差 | Look-ahead Bias | 回测时使用了当时尚未公布的信息（如用年报报告期代替公告日），导致虚假收益 | 使用实际公告日(ann_date)而非报告期(end_date) |
| 退市股票 | Delisted Stocks | 已终止上市的股票，通常业绩较差，若忽略则回测结果偏乐观 | `stock_basic`(list_status='D') |
| 报告期 vs 公告日 | Period vs Announcement Date | 报告期(end_date)是财报涵盖的时间段，公告日(ann_date)是实际发布日期 | 始终使用 ann_date 作为数据可用时间 |
| 复权信息 | Adjustment Factor | 除权除息日的处理，如果使用未来复权因子会导致回测失真 | 使用前复权，且复权因子在除权日当天更新 |
| ST标记时间 | ST Label Timing | ST标记有公告日和实施日之分，使用实施日才能避免前视 | 使用 st 接口的 imp_date（实施日期） |
| 成分股调整 | Index Reconstitution | 指数成分股定期调整，使用调整后的成分列表做历史回测会引入偏差 | 使用时点有效的成分列表 `index_weight` |
| 停牌处理 | Suspension Handling | 停牌期间无法交易，回测中若忽略停牌则可能产生虚假信号 | 结合 `suspend_d` 接口判断停牌状态 |
| 涨跌停限制 | Price Limit Constraint | 回测中若不考虑涨跌停，可能产生无法成交的虚假信号 | 结合 `stk_limit` 判断是否触及涨跌停 |

## 详细说明

### 1. 存活偏差

a. **问题本质**:
   - 使用 `stock_basic(list_status='L')` 仅获取当前上市股票
   - 已退市股票（通常是业绩差的公司）被排除在回测样本之外
   - 导致回测中的"最差股票"比实际好，策略收益被系统性高估

b. **影响程度**:
   - 学术研究表明存活偏差可使回测年化收益虚高2-5个百分点
   - 小盘股策略受存活偏差影响更大（退市公司多为小盘股）
   - A股退市率在注册制改革后显著上升，存活偏差问题更加突出

c. **防范方法**:
   ```python
   # 错误做法：只取当前上市股票
   df = pro.stock_basic(list_status='L')

   # 正确做法：取全部股票（含退市）
   df_list = pro.stock_basic(list_status='L')
   df_delist = pro.stock_basic(list_status='D')
   df_all = pd.concat([df_list, df_delist])
   ```
   - 按历史时点构建股票池：每个交易日只纳入当时仍在上市的股票
   - 结合 `list_date`（上市日期）和 `delist_date`（退市日期）做时间点过滤

### 2. 前视偏差

a. **财报数据的前视偏差**:
   - **最常见错误**: 用 `end_date='20231231'` 筛选年报数据，然后在2024年1月1日就使用这些数据
   - **实际情况**: 年报通常在次年3-4月才公告，回测中应使用 `ann_date`（公告日）
   - `income`/`balancesheet`/`cashflow` 等财务接口都包含 `ann_date` 和 `end_date` 两个字段

b. **其他常见前视偏差**:
   - **ST标记**: ST有公告日(`pub_date`)和实施日(`imp_date`)，应使用 `imp_date`
   - **指数成分**: 指数成分调整有预告期和生效期，应使用生效日
   - **复权因子**: 除权除息信息在实际除权日之前可能不完整
   - **管理层/股东变更**: 变更信息有公告日和生效日之分

c. **防范方法**:
   ```python
   # 错误做法：使用报告期
   df = pro.income(ts_code='000001.SZ', end_date='20231231')
   # 然后在2024-01-01就用这个数据做决策

   # 正确做法：使用公告日
   df = pro.income(ts_code='000001.SZ', end_date='20231231')
   # 只在 ann_date 当天或之后才能使用该数据
   df = df[df['ann_date'] <= trade_date]
   ```

### 3. 其他常见数据偏差

- **样本选择偏差**: 仅使用特定行业或市值范围的股票
- **时间窗口偏差**: 回测期恰好包含异常行情（如2015年牛市）
- **数据挖掘偏差**: 测试过多策略后选择表现最好的，需统计显著性检验
