# 卖方研究与机构行为

> 卖方研究是指券商等金融机构的研究部门对上市公司进行的分析研究，包括发布研究报告、盈利预测和投资评级等。机构行为则指基金、保险、私募等机构投资者的调研和投资活动。这些信息是市场情绪和机构动向的重要风向标。

## 核心概念

| 概念 | 说明 | 关键字段/接口 |
|------|------|---------------|
| 券商月度金股 | 各券商每月推荐的看好的股票组合 | `broker_recommend`：`broker`(券商)、`ts_code`(股票代码)、`month`(月度) |
| 卖方盈利预测 | 券商研报中对公司未来业绩的预测 | `report_rc`：`eps`(预测EPS)、`pe`(预测PE)、`roe`(预测ROE)、`rating`(评级) |
| 目标价 | 研报给出的未来股价目标 | `report_rc`：`max_price`(最高目标价)、`min_price`(最低目标价) |
| 机构调研 | 机构投资者对上市公司的调研活动 | `stk_surv`：`surv_date`(调研日期)、`rece_mode`(接待方式)、`rece_org`(调研机构) |
| 调研方式 | 机构调研的形式 | 电话会议、现场调研、网络会议、特定对象调研等 |
| 机构关注度 | 公司被机构关注的程度 | `imp_dg`(机构关注度) |

## 详细说明

### 券商月度金股（broker_recommend）

`broker_recommend` 接口提供各券商月度推荐的金股数据，需要6000积分。一般每月1-3日内更新当月数据，单次最大1000行。

数据特点：
- 每家券商每月推荐若干只看好的股票
- 数据结构简洁：`month`（月度YYYYMM）、`broker`（券商名称）、`ts_code`（股票代码）、`name`（股票简称）

使用场景：
- **共识分析**：被多家券商同时推荐的股票值得重点关注
- **券商风格追踪**：不同券商的选股风格和行业偏好
- **组合构建参考**：作为投资组合的候选池之一
- **反向指标**：当金股高度集中时需警惕一致性预期风险

### 卖方盈利预测（report_rc）

`report_rc` 接口提供券商研报的盈利预测数据，数据从2010年开始，每晚19-22点更新当日数据。单次最大3000条。120积分可试用（每天10次），8000积分可正式使用。

核心预测指标：
- **盈利预测**：`op_rt`（预测营业收入）、`np`（预测净利润）、`eps`（预测每股收益）
- **估值预测**：`pe`（预测市盈率）、`roe`（预测净资产收益率）、`ev_ebitda`（预测EV/EBITDA）
- **目标价**：`max_price`（最高目标价）、`min_price`（最低目标价）
- **评级**：`rating`（卖方评级，如买入/增持/中性/减持/卖出）
- **预测期**：`quarter`（预测报告期，如 2024Q4）

辅助字段：`report_title`（报告标题）、`org_name`（机构名称）、`author_name`（作者）、`classify`（报告分类）。

分析要点：
- 关注多家券商预测的**一致预期**（consensus），偏离一致预期的个股可能有预期差
- 券商评级普遍偏乐观，"卖出"评级极少，需注意**评级分布偏态**
- 目标价的实现率可作为评估券商研究能力的参考
- 预测值的调整方向（上调/下调）比绝对值更有信号意义

### 机构调研（stk_surv）

`stk_surv` 接口提供上市公司机构调研记录数据，单次最大100条，需要5000积分。

关键字段：
- `surv_date`：调研日期
- `rece_mode`：接待方式（电话会议、现场调研、网络会议、特定对象调研等）
- `rece_org`：参与调研的机构名称
- `fund_visitors`：机构参与人员
- `rece_place`：接待地点
- `comp_rece`：上市公司接待人员
- `content`：调研内容（默认不显示）
- `org_type`：接待公司类型

分析逻辑：
- **调研热度**：被调研次数多、参与机构多的公司通常受到更多市场关注
- **机构类型**：知名公募/保险/外资参与的调研更具参考价值
- **调研后表现**：调研密集期往往出现在业绩拐点前后
- **接待方式**：现场调研通常比电话会议传递更强烈的信号

### 数据综合应用

将三个接口结合使用可以构建更全面的分析框架：
1. 通过 `broker_recommend` 筛选券商共识看好的方向
2. 用 `report_rc` 查看具体的盈利预测和目标价
3. 结合 `stk_surv` 了解机构实地调研的动态
4. 三者交叉验证，提升判断的可靠性
