# 停复牌

> 股票因特定原因暂停交易（停牌）或恢复交易（复牌）的机制，是A股市场维护交易秩序和保护投资者利益的重要制度安排。

## 核心概念

| 概念 | 英文 | 说明 | 相关接口 |
|---|---|---|---|
| 停牌 | Suspension | 股票暂停交易，投资者无法买卖 | `suspend_d` (suspend_type='S') |
| 复牌 | Resumption | 停牌后恢复交易，投资者可正常买卖 | `suspend_d` (suspend_type='R') |
| 停牌原因 | Suspension Reason | 重大事项、资产重组、异常波动核查、信息披露等 | - |
| 日内停牌时间段 | Intraday Suspension Period | 部分停牌仅覆盖盘中特定时段（如09:30-10:00） | `suspend_d` (suspend_timing) |
| 停牌期间数据处理 | Data Handling During Suspension | 停牌期间无行情数据，K线出现缺口，复牌后可能补涨/补跌 | - |

## 详细说明

停复牌制度是交易所对股票交易进行临时性管制的重要手段。停牌的原因多种多样，常见的包括：重大事项公告（如并购重组、重大合同签署）、资产重组、股价异常波动核查（连续涨跌停后交易所要求停牌自查）、未按规定披露信息、股权分置改革相关事项等。停牌时间从数小时到数月不等，极端情况下（如退市整理）可达数年。2018年以来，交易所大幅缩短了停牌时间，鼓励不停牌或短时间停牌，以维护市场流动性。

停牌期间的数据处理是量化交易中的一个重要课题。停牌股票没有行情数据，日线K线上会出现缺口；如果停牌期间市场发生大幅波动，复牌后可能出现补涨或补跌。在计算技术指标（如均线、MACD）和回测策略时，需要特别处理停牌期间的数据缺失问题。`suspend_d` 接口提供了每日停复牌信息，包括停牌类型（S为停牌、R为复牌）和日内停牌时间段（suspend_timing），部分停牌仅覆盖盘中特定时段。

## 相关接口字段速查

**suspend_d** -- 每日停复牌信息

| 字段 | 类型 | 说明 |
|---|---|---|
| ts_code | str | 股票代码 |
| trade_date | str | 停复牌日期 |
| suspend_timing | str | 日内停牌时间段 |
| suspend_type | str | 停复牌类型：S-停牌，R-复牌 |

**接口示例**

```python
# 提取指定日期的停牌股票
df = pro.suspend_d(suspend_type='S', trade_date='20200312')

# 查询特定股票的停复牌历史
df = pro.suspend_d(ts_code='000001.SZ', start_date='20200101', end_date='20201231')
```
