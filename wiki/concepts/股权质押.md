# 股权质押

> 股权质押是上市公司股东将其持有的股份作为抵押物向金融机构（如银行、券商等质押方）借款的行为。质押期间股东仍保留投票权等股东权利，但如果股价大幅下跌触及警戒线或平仓线，质权方有权处置质押股份，可能引发强制平仓风险。

## 核心概念

| 概念 | 说明 | 关键字段/接口 |
|------|------|---------------|
| 股权质押 | 股东以股份做抵押贷款的融资行为 | `pledge_detail`：逐笔质押记录 |
| 质押比例 | 质押股份占总股本的比例 | `p_total_ratio`（本次质押占比）、`pledge_ratio`（整体质押比例） |
| 质押方/出质方 | 接受质押的金融机构/出质股份的股东 | `pledgor`（质押方）、`holder_name`（出质股东） |
| 解押 | 质押到期后解除质押的行为 | `is_release`（是否已解押）、`release_date`（解押日期） |
| 质押风险 | 高质押比例带来的平仓风险 | 质押比例过高 + 股价下跌 -> 触及平仓线 -> 强制卖出 |
| 回购 | 质押期间是否涉及回购安排 | `is_buyback`（0否 1是） |
| 质押统计 | 按股票汇总的质押情况 | `pledge_stat`：`pledge_count`（质押次数）、`unrest_pledge`/`rest_pledge` |

## 详细说明

### 质押明细数据（pledge_detail）

`pledge_detail` 接口提供逐笔质押记录，包含每次质押的详细信息：
- **质押数量**：`pledge_amount`（万股）
- **质押期限**：`start_date` 至 `end_date`
- **股东持股情况**：`holding_amount`（持股总数）、`pledged_amount`（质押总数）
- **占比信息**：`p_total_ratio`（本次质押占总股本比例）、`h_total_ratio`（持股总数占总股本比例）

必须输入 `ts_code` 参数，单次最大1000条，需要500积分。

### 质押统计数据（pledge_stat）

`pledge_stat` 接口提供按股票汇总的质押统计，按周更新：
- **质押次数**：`pledge_count`，反映股东的质押频率
- **无限售股质押**：`unrest_pledge`（万股），已流通股份的质押数量
- **限售股质押**：`rest_pledge`（万股），限售股份的质押数量
- **总股本与质押比例**：`total_share`、`pledge_ratio`

可按 `ts_code` 或 `end_date`（截止日期）查询，需要2000积分。

### 质押风险分析

股权质押的核心风险在于**平仓风险**：
1. **警戒线**：当股价跌至质押时股价的一定比例（通常150%-170%）时触发预警
2. **平仓线**：当股价跌至更低比例（通常130%-150%）时，质权方有权强制平仓
3. **连锁反应**：大量质押股份被强制卖出可能导致股价进一步下跌，形成负反馈
4. **高质押比例风险**：质押比例超过50%的公司风险较大，超过70%属于极高风险

分析建议：
- 关注 `pledge_ratio` 持续上升的公司
- 结合股价走势判断是否接近警戒线/平仓线
- 关注大股东（`holder_name`）的质押集中度
- 留意 `is_release` 的解押情况，解押可能缓解风险
