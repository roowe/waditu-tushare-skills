# 公司治理

> 公司治理涉及上市公司的组织架构、管理层构成、薪酬激励及风险警示等方面。良好的公司治理是保障股东权益、提升公司价值的重要基础，包括透明的信息披露、合理的薪酬制度、有效的风险管控等。

## 核心概念

| 概念 | 说明 | 关键字段/接口 |
|------|------|---------------|
| 上市公司基本信息 | 公司的注册信息和基本概况 | `stock_company`：法人代表、董秘、注册资本、员工人数、经营范围 |
| 管理层 | 高管的任职信息、简历、学历背景 | `stk_managers`：`title`(岗位)、`edu`(学历)、`resume`(简历)、`begin_date`/`end_date` |
| 薪酬与持股 | 管理层的报酬和持股情况 | `stk_rewards`：`reward`(报酬)、`hold_vol`(持股数) |
| ST风险警示 | 特别处理股票的风险标记 | `st`：`st_tpye`(类型)、`st_reason`(变更原因)、`st_explain`(详细原因) |
| ST触发条件 | 导致ST/*ST的具体条件 | 财务指标不达标、重大违规、持续经营能力存疑等 |
| ST撤销 | ST/*ST风险的解除条件 | 触发条件消除后可申请撤销 |
| 叠戴ST | 同时触发多种ST情形 | 如同时存在退市风险警示和其他风险警示 |

## 详细说明

### 上市公司基本信息（stock_company）

`stock_company` 接口提供上市公司的基础信息，单次提取4500条，只需120积分：
- **法人治理**：`chairman`（法人代表）、`manager`（总经理）、`secretary`（董秘）
- **资本信息**：`reg_capital`（注册资本，万元）、`setup_date`（注册日期）
- **地域信息**：`province`（省份）、`city`（城市）
- **经营信息**：`employees`（员工人数）、`main_business`（主要业务）、`business_scope`（经营范围）
- **联系信息**：`website`、`email`、`office`

可按交易所分批提取：`exchange='SSE'`（上交所）、`'SZSE'`（深交所）、`'BSE'`（北交所）。

### 管理层信息（stk_managers）

`stk_managers` 接口提供上市公司高管的详细信息，支持单个或多个股票查询，需要2000积分：
- **基本信息**：`name`（姓名）、`gender`（性别）、`birthday`（出生年月）、`national`（国籍）
- **任职信息**：`title`（岗位）、`lev`（岗位类别）、`begin_date`（上任日期）、`end_date`（离任日期）
- **教育背景**：`edu`（学历）
- **个人履历**：`resume`（个人简历，默认不显示）

### 管理层薪酬与持股（stk_rewards）

`stk_rewards` 接口提供管理层薪酬和持股数据，需要2000积分：
- `reward`：报酬（年薪）
- `hold_vol`：持股数

薪酬分析要点：
- 高薪酬+低持股 = 职业经理人模式
- 低薪酬+高持股 = 创业者/大股东管理模式
- 关注薪酬与公司业绩的匹配度

### ST风险警示（st）

`st` 接口提供ST风险警示板股票列表，单次最大1000条，需要6000积分。ST分为两类：

- **\*ST（退市风险警示）**：公司存在可能被终止上市的风险，主要触发条件包括：
  - 最近一个会计年度净利润为负且营业收入低于1亿元
  - 最近一个会计年度期末净资产为负值
  - 财务报告被出具无法表示意见或否定意见
  - 法院受理公司重整/破产清算申请

- **ST（其他风险警示）**：公司存在其他重大风险，主要触发条件包括：
  - 最近一年被出具无法表示意见或否定的内控审计报告
  - 违规对外担保或资金占用
  - 主要银行账户被冻结

**ST变更类型**（`st_tpye`）包括：ST、\*ST、叠加ST、叠加\*ST、撤销ST、撤销\*ST、从ST变为\*ST、撤销叠加\*ST等。同一股票可能多次变更，形成完整的ST状态历史。
